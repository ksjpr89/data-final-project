# 로우데이터 보유 현황 및 추가 확인 필요 항목

작성일: **2026-05-22**  
목적: 편의점 위층 공공시설 기말과제를 **상상 없이 데이터 기반으로 진행**하기 위해, 현재 보유한 원자료와 추가 확보·확인해야 할 자료를 구분한다.

## 1. 현재 보유 데이터 요약

현재 프로젝트에는 생활이동, 행정코드, 상가업소, 편의점 후보 추출 가능 자료, 인구·세대·연령, 지하철 시간대, 병원 위치, 공항 통계, 마곡 사업체, 주거 맥락 데이터가 들어 있다. 따라서 후보 행정동 수준이 아니라 **실제 편의점 주소·좌표 단위**로 내려가는 분석은 가능하다. 다만 편의점 위층의 실제 임대 가능성, 건물 층수, 엘리베이터 여부, 공실 여부는 현재 원자료만으로 확인되지 않으므로 지도·현장 확인 목록으로 분리해야 한다.

| 과제 요구사항 | 보유 상태 | 재분석에서의 사용 방식 |
|---|---:|---|
| 2015~2026 상가데이터 추이 | 부분 충족 | 기존 집계 산출물과 최신 상가업소 원자료를 사용하되, 2015~2018 원자료가 없으면 추이 해석의 한계를 명시한다. |
| 실제 편의점 주소·좌표 | 충족 가능 | 소상공인 상가업소 위치자료에서 강서구 편의점 업종을 추출해 후보 점포 지도를 만든다. |
| 인구·세대·연령 | 충족 | 행정동별 상주인구, 연령구조, 1인가구 맥락, 2030 전망 근거로 사용한다. |
| 주간상주·야간상주·유동인구 | 부분 충족 | 서울 생활이동 2026년 3월 말 4일 자료로 평일·주말과 시간대별 이용자 구조를 본다. |
| 24시간·주중주말 운영 | 부분 충족 | 생활이동 시간대와 지하철 시간대 자료로 운영모드를 설계한다. 편의점별 매출 자료는 없으므로 확정 근거로 쓰지 않는다. |
| 공항·병원·마곡 배후수요 | 충족 가능 | 병원 위치, 공항 통계, 마곡 입주기업·사업체 자료로 프로그램 수요를 교차 검증한다. |
| 지도·현장 맥락 | 추가 확인 필요 | 최종 후보 편의점 shortlist를 만든 뒤 네이버/카카오 지도, 거리뷰, 현장사진 체크리스트로 검증한다. |
| 2030 변화 전망 | 부분 충족 | 2019~2026 변화율로 단순 전망을 만들되, 장기 예측 불확실성을 함께 표기한다. |

## 2. 즉시 진행할 분석 입력 데이터

다음 단계에서는 원격 프로젝트의 대용량 원자료를 샌드박스에 복사해 분석한다. 최우선 입력은 `small_business_store_info_seoul_202603.csv`이며, 이 파일에서 **강서구 편의점 점포의 상호명, 주소, 법정동·행정동, 위도, 경도, 업종명**을 추출한다. 이후 병원·지하철·마곡·공항 관련 좌표성 데이터와 결합하여 편의점별 주변 반경 지표를 계산한다.

| 우선순위 | 원자료 | 산출 데이터 |
|---:|---|---|
| 1 | 소상공인 상가업소 서울 원자료 | `tableau_convenience_stores_gangseo.csv` |
| 2 | 병원·응급실 위치자료 | 후보 편의점 반경 병원 수, 의료 접근성 |
| 3 | 지하철역 시간대 승하차 자료 | 후보 편의점 주변 시간대 유동성 |
| 4 | 생활이동 자료 | 평일·주말, 주간·야간 이용자 구조 |
| 5 | 인구·연령·세대 자료 | 프로그램 이용자층과 2030 변화 전망 |
| 6 | 마곡 입주기업·사업체 자료 | 직장인·교대근무·업무지원 수요 |

## 3. 추가 확인 필요 항목

최종 PPT에서 확정적으로 말하면 안 되는 항목은 별도 표로 관리한다. 이 항목들은 데이터 분석 결과를 만든 뒤 지도 검색 또는 사용자의 현장 확인으로 보완해야 한다.

| 확인 필요 항목 | 이유 | 확인 방법 |
|---|---|---|
| 후보 편의점 건물의 위층 존재 여부 | 과제 대상은 ‘편의점 위층’이므로 건물 조건이 핵심이다. | 네이버지도·카카오맵 거리뷰, 건물 사진, 현장 방문 |
| 위층 공실 또는 공공시설 전환 가능성 | 데이터상 편의점이 있어도 공간 확보가 불확실하다. | 부동산 정보, 현장 관찰, 건물 외관 확인 |
| 장애인·노약자 접근성 | 공공시설 운영에는 엘리베이터, 계단, 출입구가 중요하다. | 거리뷰·현장 사진 |
| 실제 주변 경쟁 공공시설 | 과제 지침상 동일 경쟁 프로그램 영향은 필수 고려 대상은 아니나, 발표 설득력 보강용으로만 확인한다. | 지도 검색 |
| 시간대별 실제 체감 혼잡 | 생활이동·지하철 자료는 주변 평균이므로 점포 앞 체감과 다를 수 있다. | 현장 방문 또는 지도 리뷰·거리뷰 |

## 4. 결론

현재 보유 데이터만으로도 ‘행정동 추천’이 아니라 **강서구 실제 편의점 후보지별 비교 분석**으로 내려갈 수 있다. 다만 최종 후보를 한 곳으로 확정하려면 상가업소 좌표 기반의 1차 shortlist 이후 지도 검색과 건물 조건 검증이 반드시 필요하다. 따라서 다음 단계는 편의점 위치 원자료 추출, Tableau용 편의점 지도 CSV 생성, 후보 편의점 반경 분석으로 진행한다.
