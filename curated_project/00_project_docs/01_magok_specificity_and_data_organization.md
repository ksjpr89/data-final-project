# 01. 마곡 특수성 논의 프레임과 데이터 정리 방향

## 1. 이번에는 결론이 아니라 질문부터 세운다

이번 과제는 “마곡이 사람이 많은 곳인가?”를 묻는 분석이 아니다. 우리가 처음 세운 문제의식은 **마곡이 서울의 다른 역세권과 다른 특수한 수요 구조를 갖는가**였고, 따라서 분석의 출발점도 유동인구 순위가 아니라 마곡이라는 지역을 특별하게 만드는 구조적 조건이어야 한다.

> 핵심 질문은 “마곡에 수요가 있나?”가 아니라, **“마곡의 어떤 특수성이 어떤 유형의 체류·이용 수요로 이어질 수 있나?”**이다.

이 질문을 기준으로 보면 마곡의 특수성은 크게 다섯 개 축으로 나눌 수 있다. 첫째, 김포공항과 가까운 **공항 이동 거점성**이다. 둘째, 이대서울병원과 응급의료센터가 만드는 **병원·보호자·교대근무 수요**이다. 셋째, 마곡산업단지와 대기업 연구소가 만든 **R&D 업무지구성**이다. 넷째, 강서구 자체가 서울 평균보다 큰 1인가구 규모를 가진다는 **1인가구 기반 수요**이다. 다섯째, 10년 전과 비교했을 때 마곡이 빈 땅·개발지에서 기업과 주거가 결합된 도시로 바뀐 **도시 변화성**이다.

## 2. 마곡 특수성 5개 축

| 축 | 우리가 확인하려는 특수성 | 지금 확보한 근거 | 현재 해석 수준 |
|---|---|---|---|
| 공항 축 | 김포공항 접근성이 새벽·심야 이동, 탑승 전후 대기, 공항 관련 종사자·방문자 수요를 만들 수 있는가 | 김포공항 2025년 총여객 2,296만 명, 국내선 비중 80.67% | 근거는 있으나 항공 종사자 거주지·단기 체류 자료는 추가 필요 |
| 병원 축 | 이대서울병원과 응급의료센터가 보호자 대기, 교대근무자, 야간·긴급 체류 수요를 만들 수 있는가 | 이대서울병원 1,014병상, 2024년 거점 지역응급의료센터 지정 | 병원 규모와 기능은 명확하나 보호자 체류 실태 자료는 추가 필요 |
| 기업/R&D 축 | 마곡산업단지 기업 유입이 점심·퇴근 후·야근·출장·외부 방문 수요를 만들 수 있는가 | 2025년 마곡사업장 근무인원 39,966명, R&D 인력 18,301명 | 가장 강한 특수성 축이며, 마곡이 10년 전과 달라진 핵심 근거 |
| 1인가구 축 | 강서구 및 마곡 인접 생활권의 1인가구 규모가 서울 평균보다 큰가 | 강서구 2024년 1인가구 106,748가구, 서울 자치구 평균의 1.61배 | 강서구 단위 근거는 강하지만 마곡 법정동 단위는 행정동 근사 필요 |
| 도시 변화 축 | 마곡이 과거와 다른 기업·주거 복합 생활권으로 전환되었는가 | 마곡산업단지 기업·고용·R&D 지표, 인구·가구 변화 지표 | 정성적으로 매우 중요하며, 과제 스토리의 중심축으로 활용 가능 |

## 3. 지금 단계에서 잡아야 할 논리

마곡은 단순히 **역 주변 유동인구가 많은 상권**으로 설명하면 안 된다. 그렇게 접근하면 홍대, 강남, 여의도, 신림 같은 기존 고유동 지역과 비교했을 때 마곡만의 설득력이 약해진다. 대신 마곡은 서로 다른 시간대와 목적을 가진 수요가 겹치는 지역으로 봐야 한다.

공항 수요는 이른 아침과 늦은 밤, 짐을 들고 이동하는 사람, 국내선 반복 이용자, 출장자, 공항 종사자라는 특징을 가진다. 병원 수요는 응급실 보호자, 입원 환자 가족, 교대근무 의료진, 병원 주변에서 짧게 쉬거나 대기해야 하는 사람으로 나타난다. 기업/R&D 수요는 정규 출퇴근 인구뿐 아니라 야근, 프로젝트 방문, 외부 협력사 미팅, 출장자 체류로 이어질 수 있다. 1인가구 수요는 주거 주변의 소형 소비, 간편식, 세탁, 무인 서비스, 짧은 휴식 공간, 저녁 시간대 생활편의 수요로 연결된다.

따라서 마곡의 핵심 인사이트는 **“많은 사람이 지나간다”가 아니라 “서로 다른 이유로 잠깐 머물러야 하는 사람이 반복적으로 발생한다”**에 가깝다. 과제의 수요 분석도 이 방향으로 세워야 한다.

## 4. 파일이 복잡한 문제의 원인

현재 폴더가 복잡해진 이유는 분석을 여러 번 방향 전환하면서 초기 상권 분석 자료, 확장 분석 자료, 마곡 특수성 재분석 자료, 원자료, 정제자료, 차트, 보고서가 한 프로젝트 안에 함께 쌓였기 때문이다. 파일 수는 약 136개이고, 특히 `reanalysis/outputs` 안에 정제 데이터·최종 보고서·중간 검증표·Tableau용 데이터가 섞여 있어 사용자가 직접 찾기 어렵다.

| 현재 위치 | 문제점 | 개선 방향 |
|---|---|---|
| `expanded_inputs/` | 초기 확장 분석 원자료가 무엇을 의미하는지 설명이 부족함 | `01_data/raw_external/`로 이동하거나 카탈로그에서 설명 |
| `outputs/` | 초기 후보지 분석 산출물과 최종 분석 산출물이 혼재됨 | `04_outputs/old_candidate_analysis/`와 `04_outputs/final_magok_analysis/`로 구분 |
| `reanalysis/raw/` | 공식 PDF와 엑셀 원자료는 좋지만 설명 파일이 분산됨 | 원자료별 출처·용도·갱신일을 데이터 카탈로그에 등록 |
| `reanalysis/outputs/` | 최종 보고서, 중간표, Tableau 파일, 검증표가 한 폴더에 섞임 | 보고서·표·차트·대시보드용 데이터를 하위 폴더로 분리 |
| 루트 폴더 | 스크립트와 메모가 일부 상위에 남아 있음 | `03_scripts/`와 `docs/`로 역할 분리 |

## 5. 추천 폴더 구조

바로 이해 가능한 구조로 바꾸려면 아래처럼 **번호를 붙인 폴더 구조**가 가장 좋다. 이 방식은 다른 PC에서 클론했을 때도 읽는 순서가 자연스럽고, 과제 제출물과 원자료를 구분하기 쉽다.

| 새 폴더 | 역할 | 예시 파일 |
|---|---|---|
| `00_project_docs/` | 과제 기획서, 작업 메모, README 보조 문서 | 마곡 특수성 논의 프레임, 데이터 카탈로그 |
| `01_raw_data/` | 원본 파일, 다운로드 자료, PDF, 엑셀, HTML | 공항 통계 원문, 강서구 1인가구 엑셀, 마곡산단 PDF |
| `02_processed_data/` | 분석에 바로 쓰는 정제 CSV·JSON | 김포공항 연도별 통계, 1인가구 정제표, 인구 비교표 |
| `03_scripts/` | 수집·정제·분석·시각화 코드 | `collect_*.py`, `analyze_*.py`, `create_*.py` |
| `04_outputs/` | 최종 표, 차트, 보고서, PPT 준비 자료 | 최종 보고서, synthesis tables, figures |
| `05_tableau_or_dashboard/` | Tableau 또는 대시보드용 데이터셋 | 편의점 후보지, 공공시설 공급 공백 데이터 |
| `99_archive/` | 예전 방향의 중간 산출물 보관 | 초기 확장 분석, 폐기한 후보지 분석 |

## 6. 데이터 카탈로그로 해결할 수 있는 것

폴더명만 바꿔도 좋아지지만, 진짜 중요한 것은 **데이터 카탈로그**를 만드는 것이다. 데이터 카탈로그에는 각 파일이 무엇이고, 어디서 왔고, 어떤 분석에 쓰였는지를 한 줄씩 적는다. 그러면 파일을 직접 열어보기 전에도 필요한 자료를 찾을 수 있다.

| 카탈로그 항목 | 설명 |
|---|---|
| 파일명 | 실제 파일 이름 |
| 분류 | 원자료, 정제자료, 분석결과, 차트, 보고서, 스크립트 |
| 주제축 | 공항, 병원, 1인가구, 기업/R&D, 후보지, 공공시설 등 |
| 출처 | 서울 열린데이터광장, 한국공항공사, 마곡산업단지, 직접 작성 등 |
| 사용처 | 어떤 보고서·표·차트에 사용됐는지 |
| 중요도 | 최종 제출 필수, 참고, 보관용, 폐기 후보 |
| 비고 | 주의사항, 행정동 근사, 추가 검증 필요 등 |

## 7. 내가 제안하는 진행 순서

우선 지금 당장 실제 파일을 이동하기보다는, 먼저 **정리 설계안과 데이터 카탈로그 초안**을 만드는 것이 안전하다. 파일 이동은 경로가 바뀌면 기존 스크립트의 상대경로가 깨질 수 있기 때문이다. 따라서 1단계로 카탈로그를 만들고, 2단계로 핵심 파일만 새 구조에 복사하고, 3단계로 스크립트 경로를 수정하는 순서가 좋다.

| 단계 | 작업 | 이유 |
|---|---|---|
| 1단계 | 현재 파일 전체를 카탈로그화 | 무엇을 보존·이동·폐기할지 판단 가능 |
| 2단계 | 마곡 특수성 관련 핵심 파일만 우선 분류 | 과제 방향과 직접 관련 있는 자료부터 정돈 |
| 3단계 | 새 폴더 구조를 만들고 파일을 복사 | 기존 경로를 깨뜨리지 않고 안전하게 정리 가능 |
| 4단계 | README와 데이터 카탈로그 업데이트 | 다른 PC에서 이어서 작업 가능 |
| 5단계 | 정리 완료 후 Git 커밋 | 변경 이력을 남겨 되돌릴 수 있음 |

## 8. 다음 대화에서 같이 정하면 좋은 질문

다음으로는 마곡 특수성을 과제의 핵심 주장으로 어떻게 표현할지 정하면 된다. 나는 현재 기준으로 다음 문장이 가장 적절하다고 본다.

> **마곡은 단순 고유동 역세권이 아니라, 김포공항·이대서울병원·마곡산업단지·강서구 1인가구가 중첩되면서 이동 전후 대기, 보호자 체류, R&D 근무·방문, 소형 생활편의 수요가 동시에 발생하는 복합 체류 생활권이다.**

이 문장을 중심 주장으로 놓고, 이후에는 각 축을 하나씩 검증하면 된다. 먼저 공항 축을 볼지, 병원 축을 볼지, 기업/R&D 축을 볼지, 1인가구 축을 볼지 순서를 정하면 된다.
